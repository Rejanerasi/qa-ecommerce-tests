# 🧪 QA E-commerce Tests

Repositório com simulação de testes de software em um sistema de e-commerce.

---

## 🎯 Objetivo

Demonstrar práticas de Quality Assurance (QA), incluindo:

- Criação de casos de teste
- Identificação e documentação de bugs
- Execução de retestes
- Organização de testes em ambiente versionado

---

## 📁 Estrutura do Projeto

qa-ecommerce-tests/
 ├── test-cases/
 │    └── login.md
 ├── bug-reports/
 │    └── login-bug.md
 └── README.md

---

## 🧪 Tipos de Testes

- Testes funcionais
- Testes exploratórios
- Retestes de bugs

---

## 🐞 Gestão de Bugs

Os bugs são documentados contendo:

- Descrição
- Passos para reprodução
- Resultado esperado
- Resultado atual
- Severidade

---

## 🔄 Fluxo de Trabalho

1. Criação de casos de teste  
2. Execução dos testes  
3. Identificação de bugs  
4. Registro via issue  
5. Reteste após correção  
6. Validação e fechamento  

---

## 🛠️ Ferramentas Utilizadas

- GitHub (controle de versão)
- Markdown (documentação)
  
---

## 📌 Observações

Este projeto simula um ambiente real de QA, aplicando boas práticas de organização e documentação de testes.
